### 1. Claude Code v2.1.101 — Enterprise & Team Features Drop
**What it does:** Adds `/team-onboarding` (generates a teammate ramp-up guide from your local Claude Code usage patterns), OS CA certificate store trust (enterprise TLS proxy support out of the box), dynamic `/loop` (Claude auto-determines tick frequency without explicit intervals), plus UltraPlan and Focus Mode refinements.

### 2. Claude Code v2.1.97 — Focus View & Critical MCP Memory Leak Fix
**What it does:** Introduces **Focus View** (`Ctrl+O` in NO_FLICKER mode) — a stripped-down display showing only prompt, one-line tool summaries with diffstats, and final response. Fixes an MCP HTTP/SSE memory leak that was silently accumulating ~50MB/hour (~400MB in an 8-hour session). Also adds 429 exponential backoff (fixes rapid retry burns), Bash permissions hardening, and `/resume` fixes.

**Why it matters:** If you run long Claude Code sessions with MCP servers, you were leaking significant memory and didn't know it. Focus View is a genuine quality-of-life win for developers who found the default output too noisy during multi-step tasks. The 429 backoff fix stops wasted retries during rate limiting.

**Verification:** ✅ Official — [GitHub releases](https://github.com/anthropics/claude-code/releases), [ton-technotes.com](https://ton-technotes.com/en/blog/2026-04-09-claude-code-weekly-update-v2197/), [claude-news.today](https://claude-news.today/en/briefings/briefing-2026-04-09/), [newclawtimes.com](https://newclawtimes.com/articles/claude-code-v2197-focus-view-bash-hardening-permissions-update)

### 4. Shopify AI Toolkit — Claude Code Gets a First-Class Commerce Plugin
**What it does:** Shopify released a free, MIT-licensed AI Toolkit (April 9) that integrates directly with Claude Code as a plugin. It provides live Shopify documentation access, real-time GraphQL/Liquid schema validation, and the ability to execute store mutations (product updates, discounts, inventory) directly from Claude Code. Supports 16 skills across Admin API, Storefront, Liquid, Hydrogen, and Polaris.

**Why it matters:** This is the most polished third-party Claude Code plugin integration to date. If you build on Shopify, Claude Code now has validated, up-to-date API knowledge instead of relying on training data. Install is two commands. **Caveat:** store mutations are live with no undo — query state first.

**Install:**
```
/plugin marketplace add Shopify/shopify-ai-toolkit
/plugin install shopify-plugin@shopify-ai-toolkit
```

**Tip:** Set `OPT_OUT_INSTRUMENTATION=true` if working with proprietary code — validation sends payloads by default.

**Verification:** ✅ Official (Shopify) — [awesomeagents.ai](https://awesomeagents.ai/news/shopify-ai-toolkit-mcp-agents/), [zenvanriel.com](https://zenvanriel.com/ai-engineer-blog/shopify-ai-toolkit-guide-claude-code-cursor/), [fudge.ai](https://www.fudge.ai/guides/shopify-ai-toolkit-claude-code-setup/), [Shopify YouTube demo](https://www.youtube.com/shorts/u2_UOBcbMa8)

## 🔧 Useful Tips & Tricks

### Compounding Knowledge Loop via Hooks
Use Claude Code hooks to automatically capture session learnings (errors, corrections, design decisions) into structured storage (CLAUDE.md, Obsidian vaults). Over time, this builds a project-specific wiki that improves agent responses across sessions. A YouTube walkthrough implementing Karpathy's autoresearch pattern got significant traction this week.
- **Source:** [mindstudio.ai](https://www.mindstudio.ai/blog/compounding-knowledge-loop-claude-code/), [YouTube](https://www.youtube.com/watch?v=7huCP6RkcY4)
- **Status:** Community technique using official features

### Token Cost Optimization — Practical Strategies
Multiple posts this week converged on the same playbook:
- Use `/compact` aggressively and `/clear` between unrelated tasks
- Route subagents to Haiku/Sonnet for exploration; reserve Opus for complex reasoning
- Use `/cost` command (added in v2.1.92) to monitor spend in real-time
- Batch related tasks in single sessions to avoid context reload costs
- One engineer reported cutting monthly spend ~50% with these techniques
- **Source:** [samuellawrentz.com](https://samuellawrentz.com/blog/claude-code-token-optimization/), [claudecode-lab.com](https://claudecode-lab.com/en/blog/claude-code-token-optimization/), [mindstudio.ai](https://www.mindstudio.ai/blog/save-tokens-claude-code-opus-plan-mode/)

### UltraPlan for Complex Multi-File Work
`/ultraplan` offloads planning to Anthropic's cloud with multi-agent codebase exploration (up to 30 min of "thinking time"). Review plans in browser with inline comments, then execute remotely (pushes branch + PR) or teleport back to local CLI. ~2x faster than local planning in community tests. Best for migration tasks and large refactors where you want to audit the "blast radius" before executing.
- **Source:** [dev.to](https://dev.to/aabyzov/claude-code-ultraplan-why-the-workflow-matters-more-than-the-hype-3p2n), [stevekinney.com](https://stevekinney.com/writing/claude-ultraplan), [uxplanet.org](https://uxplanet.org/the-best-way-to-plan-work-with-claude-code-58e336a3b0ba)
