# Claude Code Weekly — May 5–11, 2026

**This was the biggest Claude Code week of the year.** Anthropic held its "Code w/ Claude 2026" developer conference on May 6 in San Francisco, shipping major platform features. Meanwhile, serious security disclosures hit, and the community surfaced a costly caching bug.

---

## 🚀 What Actually Mattered

### 1. Claude Code Routines — Async, Autonomous Coding Workflows
**What it does:** Routines are saved Claude Code configurations — a prompt, repos, environment, connectors — that run autonomously on Anthropic's cloud infra. Triggered by **cron schedules**, **webhooks** (GitHub events, CI failures, monitoring alerts), or API calls. They execute full Claude Code sessions without you present, then output commits, PRs, Slack messages, or file artifacts.

**Why it matters:** This is the shift from interactive coding assistant to **background coding agent**. Set up a routine to auto-triage your backlog weekly, auto-review PRs on open, or auto-fix failing CI. Boris Cherny framed it as: "going forward a lot of code is going to be written in an async way." Early adopters report 80% time savings on repetitive tasks.

**Status:** Live now via Claude Desktop (Code tab → Routines → New Routine) and API.

- **Verification:** Official — announced at keynote, documented at code.claude.com
- **Sources:** [Simon Willison liveblog](https://simonwillison.net/2026/May/6/code-w-claude-2026/), [Lenny's Newsletter](https://www.lennysnewsletter.com/p/code-with-claude-the-5-biggest-updates), [claudedirectory.org guide](https://www.claudedirectory.org/blog/claude-code-routines-guide), [Engincan Veske deep-dive](https://engincanveske.substack.com/p/claude-routines-automate-the-checks)

### 2. Managed Agents: Dreaming, Outcomes, Multi-Agent Orchestration
Three new capabilities for Claude Managed Agents, all announced May 6:

| Feature | What It Does | Status |
|---------|-------------|--------|
| **Dreaming** | Agent reviews up to 100 past session transcripts, extracts patterns, merges/updates its memory store. Runs as a scheduled overnight process. Auto-apply or manual review modes. | Research preview (request access) |
| **Outcomes** | Define success via a rubric. A separate grader agent (isolated context) evaluates output, gives feedback, iterates until criteria met. Supports webhook notifications. | Public beta |
| **Multi-Agent Orchestration** | Lead agent delegates to specialist sub-agents with custom models/prompts/tools. They work in parallel on shared filesystems with persistent events. | Public beta |

**Why it matters:** Dreaming ends "goldfish memory" across sessions — Harvey reported 6x task completion improvement. Outcomes add self-correction without human babysitting. Multi-agent orchestration lets you shard context across specialists instead of stuffing everything into one window.

- **Verification:** Official — [9to5Mac](https://9to5mac.com/2026/05/07/anthropic-updates-claude-managed-agents-with-three-new-features/), [SD Times](https://sdtimes.com/ai/new-in-claude-managed-agents-dreaming-outcomes-and-multiagent-orchestration/), [Ken Huang analysis](https://kenhuangus.substack.com/p/claude-agents-can-now-dream-how-ai)

### 4. CVE-2026-39861: Sandbox Escape via Symlink + Related Vulns
**What happened:** Claude Code's sandbox didn't prevent sandboxed processes from creating symlinks pointing outside the workspace. When the unsandboxed Claude Code process wrote to a path through such a symlink, it followed it — enabling **arbitrary file writes outside the workspace** without user confirmation.

**Related vulnerabilities also disclosed:**
- Git worktree `commondir` trust bypass (malicious repo executes hooks from `.claude/settings.json`)
- Piped `sed` command bypass (evades file write restrictions)
- `.claude/settings.json` protection failure (sandboxed creation of persistent hooks)

**Why it matters:** If you clicked "ok" on a malicious command, your filesystem was exposed. The HN thread notes a researcher found this in August 2025 but it was excluded from Anthropic's VDP at the time.

**Fix:** All patched via auto-update. Manual users: `npm update @anthropic-ai/claude-code`.

- **Verification:** Official — [GitLab Advisory Database](https://advisories.gitlab.com/pkg/npm/@anthropic-ai/claude-code/), [Hacker News discussion](https://news.ycombinator.com/item?id=48057842)

### 5. "TrustFall" — Supply Chain Attack via Malicious Repos + MCP
**What it is:** Adversa.AI demonstrated that attackers can create repos containing malicious MCP server configurations. When Claude Code auto-clones and trusts such a repo, it can execute arbitrary code, exfiltrate env vars, deploy keys, and signing certificates.

**Why it matters:** Especially dangerous when Claude Code runs in CI/CD pipelines — a compromised dependency becomes a full supply chain attack. This is a class of attack, not a single bug.

- **Verification:** Official research — [SecurityWeek](https://www.securityweek.com/ai-coding-agents-could-fuel-next-supply-chain-crisis/), [Dark Reading](https://www.darkreading.com/application-security/trustfall-exposes-claude-code-execution-risk)

### 7. Prompt Cache Regression — Up to 20x Cost Increase on Resumed Sessions
**What happened:** A regression in Claude Code causes resumed sessions to bypass prompt caching, re-evaluating the full context on every turn. Heavy users on Max plans report **up to 20x token cost spikes** and premature weekly limit depletion.

**Community fix:** [cnighswonger/claude-code-cache-fix](https://github.com/cnighswonger/claude-code-cache-fix) — adds explicit `cache_control` markers and session state persistence. Related issue: [anthropics/claude-code#57699](https://github.com/anthropics/claude-code/issues/57699).

**Why it matters:** If you resume sessions frequently on Opus, you may be burning tokens at dramatically higher rates without realizing it.

- **Verification:** Community-reported — GitHub issue + community fix. No official Anthropic acknowledgment found yet.

## 🔧 Useful Tips & Tricks

### Ask for HTML, Not Markdown
Thariq Shihipar (Claude Code team at Anthropic) published a piece arguing that requesting **HTML output** from Claude instead of Markdown produces dramatically richer results — color-coded diffs, collapsible sections, SVG diagrams, interactive elements. Try adding "output as an interactive HTML page" to your next complex prompt.

- **Source:** [Simon Willison's link](https://simonwillison.net/2026/May/8/unreasonable-effectiveness-of-html/), [GogoAI analysis](https://gogoai.xin/article/why-html-beats-markdown-for-claude-output)
- **Verification:** Official (Anthropic employee), widely validated by community

### Routines for PR Auto-Review
Damian Galarza published a practical walkthrough of using Claude Code Routines to auto-maintain PR evolution numbers — triggered on GitHub PR events, runs team checklists, posts inline comments.

- **Source:** [damiangalarza.com](https://www.damiangalarza.com/posts/2026-05-06-claude-code-routine-auto-maintain-pr-evolution-numbers/)
- **Verification:** Community-reported, uses official Routines feature

### Watch Your Resumed Session Costs
If you're on a Max plan and resuming long sessions, check your actual token consumption via `ccusage`. The cache regression (item #7 above) may be silently draining your limits. Consider the community fix or starting fresh sessions for now.

- **Source:** [GitHub issue #57699](https://github.com/anthropics/claude-code/issues/57699)
- **Verification:** Community-reported

### Security Hygiene Reminders
Given CVE-2026-39861 and TrustFall this week:
- **Update Claude Code** if you're on manual installs
- **Don't auto-trust cloned repos** — review `.claude/settings.json` and MCP configs before accepting
- **Be cautious in CI/CD** — audit any Claude Code automation touching untrusted repos
- **Avoid "Claude Code" downloads from Google Ads** — active malvertising campaign pushing Mac malware via fake Claude download pages ([BleepingComputer](https://www.bleepingcomputer.com/news/security/hackers-abuse-google-ads-claudeai-chats-to-push-mac-malware/))

## 🧭 Bottom Line

**Most important update:** **Claude Code Routines.** This is the feature that changes how you use Claude Code — from interactive tool to autonomous background agent. If you do anything repetitive (PR reviews, CI fixes, backlog triage), set up a Routine this week.

**Most useful workflow improvement:** **Doubled rate limits + no peak-hour throttling.** Immediately noticeable for anyone who was hitting walls during heavy coding sessions.

**Most important model-related change:** **None this week.** No new model was announced. The improvements are all platform/infrastructure — more capacity, better agent orchestration, smarter memory. The "infinite context" tease is forward-looking only.

**Overhyped / low-signal:** The "infinite context window" framing in YouTube thumbnails and some coverage is misleading — it was a keynote tease about combining memory with context, not a shipped capability. Also, the leaked feature flags story (44 hidden flags) generated clicks but most flags are internal/experimental with no practical developer action.

**What developers should try now:**
1. **Set up your first Routine** — start with something simple like auto-reviewing PRs against your team's checklist
2. **Check your Claude Code version** — make sure you're patched for CVE-2026-39861 (`claude --version`, update if needed)
3. **Try the HTML output trick** on your next complex analysis or code review prompt
4. **If on Max plan:** monitor your token spend on resumed sessions — the cache regression is real
