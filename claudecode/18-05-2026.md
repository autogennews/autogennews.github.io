## 🚀 What Actually Mattered

### 1. Opus 4.7 Fast Mode Released & Becomes Claude Code's Fast-Mode Default

**What it does:** Anthropic released Claude Opus 4.7 Fast on May 12 — same Opus 4.7 intelligence with ~2.5× faster output token generation. Two days later, Claude Code **v2.1.142** made Opus 4.7 the default model for fast mode, replacing Opus 4.6.

**Why it matters:** If you use fast mode in Claude Code (and most power users do), your sessions are now running a meaningfully better model by default. The speed bump is real for interactive coding. However, Opus 4.7 Fast is priced at **$30/$150 per MTok** (input/output) via API — significantly more expensive than standard Opus 4.7 ($5/$25). Subscription users won't see this directly, but API/programmatic users will. You can pin back to Opus 4.6 with `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1` if needed.

**Verification:** Official — confirmed via Releasebot changelog, OpenRouter listing, Windsurf blog, TLDR newsletter.
**Sources:** [Releasebot](https://releasebot.io/updates/anthropic/claude-code), [OpenRouter](https://openrouter.ai/anthropic/claude-opus-4.7-fast), [Windsurf blog](https://windsurf.com/blog/opus-4.7-fast), [Pravin Kumar blog](https://www.pravinkumar.co/blog/claude-code-opus-4-7-fast-mode-default-2026)

### 2. Agent View — CLI Dashboard for Multi-Session Management (Research Preview)

**What it does:** Launched May 11 as a Research Preview, Agent View (`claude agents`) gives you a single CLI dashboard showing all your active Claude Code sessions — running, waiting for input, backgrounded, or completed. You can peek at status, reply inline, or jump into full transcripts. Background sessions via `claude --bg` or `/bg` feed into this view.

**Why it matters:** This is the biggest workflow change of the week. Anyone running parallel agents, background tasks, or multi-repo work no longer needs tmux grids or tab-switching. The community response was massive — multiple YouTube walkthroughs within 48 hours, blog coverage from at least 5 outlets. This is clearly what developers were waiting for.

**Verification:** Official — blogged at [claude.com/blog/agent-view-in-claude-code](https://claude.com/blog/agent-view-in-claude-code), documented at [code.claude.com/docs/en/agent-view](https://code.claude.com/docs/en/agent-view).
**Sources:** [Anthropic blog](https://claude.com/blog/agent-view-in-claude-code), [Times of AI](https://www.timesofai.com/industry-insights/agent-view-in-claude-code/), [YouTube coverage](https://www.youtube.com/watch?v=DbO5IyfPwkE)

### 3. Routines — Scheduled & Event-Driven Claude Code Automation

**What it does:** Routines let you configure Claude Code workflows that run on schedules, via API calls, or in response to external events (e.g., GitHub webhooks). They run on Anthropic's cloud infrastructure — no cron jobs or local servers needed. A routine includes a prompt, repo access, and connected tools/services.

**Why it matters:** This turns Claude Code from an interactive tool into a persistent automation layer. Use cases already being discussed: automated PR triage, documentation drift detection, CI failure analysis, cross-language SDK sync. Combined with Agent View, this makes "fleet of agents" workflows practical for individual developers.

**Verification:** Mixed — covered by [InfoQ](https://www.infoq.com/news/2026/05/anthropic-routines-claude/) and [daily.dev](https://app.daily.dev/posts/anthropic-introduces-routines-for-claude-code-automation-0cm6saulh); official docs not directly confirmed in search but consistent with desktop changelog mention of `update_scheduled_task`.
**Sources:** [InfoQ](https://www.infoq.com/news/2026/05/anthropic-routines-claude/), [daily.dev](https://app.daily.dev/posts/anthropic-introduces-routines-for-claude-code-automation-0cm6saulh)

### 4. Anthropic Publishes Quality Postmortem — Explains 6 Weeks of Complaints

**What it does:** Anthropic published an engineering postmortem (May 14) explaining why Claude Code quality felt degraded for ~50 days. Root cause: **three overlapping product-layer bugs**, not model changes:
1. A **system prompt change** that reduced reasoning verbosity
2. A **caching bug** that cleared the model's reasoning context on every turn instead of only after idle periods
3. A **tokenizer regression** that degraded output for a subset of traffic

All three were fixed by April 20 in v2.1.116.

**Why it matters:** This is important context. If you experienced inconsistent Claude Code quality in March–April 2026, it wasn't the model — it was product bugs. The postmortem is unusually transparent and worth reading. It also explains why different users saw wildly different symptoms.

**Verification:** Official — published on [anthropic.com/engineering](https://www.anthropic.com/engineering), covered by [InfoQ](https://www.infoq.com/news/2026/05/anthropic-claude-code-postmortem/) and [Remio](https://www.remio.ai/post/anthropic-claude-code-was-broken-for-50-days-the-postmortem-s-real-lesson-is-not-the-bugs).
**Sources:** [InfoQ](https://www.infoq.com/news/2026/05/anthropic-claude-code-postmortem/), [Anthropic Engineering](https://www.anthropic.com/engineering)

### 5. Billing Split Announced — Programmatic Usage Gets Separate Credit Pool (June 15)

**What it does:** Starting **June 15, 2026**, Anthropic is splitting subscription billing into two pools:
- **Interactive** (terminal/IDE Claude Code, Claude.ai chat) → stays on existing subscription
- **Programmatic** (`claude -p`, GitHub Actions, Agent SDK, third-party tools like Zed) → new monthly Agent SDK credit pool at API rates

Credit by plan: **Pro ~$20/mo**, **Max 5x ~$100/mo**, **Max 20x ~$200/mo**. Unused credits don't roll over.

**Why it matters:** This is the most consequential upcoming change for power users. If you rely heavily on `claude -p` in CI pipelines, GitHub Actions, or third-party integrations, your effective budget is about to shrink dramatically. Axios framed it as "Anthropic tightens Claude limits as OpenAI courts agent users." Zed published a [blog post](https://zed.dev/blog/anthropic-subscription-changes) specifically addressing the impact on their users.

**Verification:** Official — confirmed by [Zed](https://zed.dev/blog/anthropic-subscription-changes), [InfoWorld](https://www.infoworld.com/article/4171274/anthropic-puts-claude-agents-on-a-meter-across-its-subscriptions.html), [Axios](https://www.axios.com/2026/05/14/anthropic-claude-price-openai-tokens), [BuildThisNow migration guide](https://www.buildthisnow.com/blog/guide/mechanics/claude-billing-change-june-2026).

### 7. Claude Code v2.1.142 & v2.1.143 — Agent Flags, Plugin Hardening, Worktree Controls

**What they do (highlights beyond the above):**
- **New `claude agents` flags:** `--model`, `--effort`, `--permission-mode`, `--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir`, `--dangerously-skip-permissions` — much more control over background agent sessions
- **Plugin dependency enforcement:** `claude plugin disable` now refuses if another plugin depends on it; `claude plugin enable` force-enables transitive deps
- **`worktree.bgIsolation: "none"`** — lets background sessions edit the working copy directly without creating an isolated worktree (useful for repos where worktrees are impractical)
- **Hooks guardrail:** hooks now stop after 8 consecutive blocks with a warning, preventing infinite loops
- **`/plugin` shows projected context cost** before enabling
- **Desktop:** OS notifications on session completion, scheduled tasks can self-modify via `update_scheduled_task`, auto-reload MCP tools on config change

**Why it matters:** The agent flags are the sleeper hit here. Being able to set `--model` and `--effort` per background agent means you can run cheap Sonnet agents for simple tasks and Opus agents for complex ones, all from Agent View. The worktree opt-out is a pragmatic fix for monorepos and non-standard git setups.

**Verification:** Official — [GitHub releases](https://github.com/anthropics/claude-code/releases), [Releasebot](https://releasebot.io/updates/anthropic/claude-code), [Claude Code desktop changelog](https://code.claude.com/docs/en/desktop-changelog).

## 🧭 Bottom Line

**Most important update:** The **June 15 billing split**. It hasn't happened yet, but it was announced this week and will materially change how power users budget for programmatic Claude Code usage. Start auditing now.

**Most useful workflow improvement:** **Agent View** (`claude agents`). If you run more than one Claude Code session at a time, this is an immediate quality-of-life upgrade. Combined with the new `--model` and `--effort` flags, it makes multi-agent workflows genuinely manageable.

**Most important model-related change:** **Opus 4.7 Fast becoming the default fast-mode model**. You're now getting better output by default. If you're on API billing, watch the cost impact.

**Overhyped / low-signal:** The Codex-vs-Claude-Code comparison pieces flooding the internet this week are mostly benchmark theater. Both tools are good. Pick based on your stack and workflow, not SWE-bench decimal points.

**What developers should try now:**
1. Run `claude agents` and try dispatching 2–3 background tasks across different repos
2. Set up a Routine for something you do weekly (dependency updates, doc checks, test coverage scans)
3. Audit any `claude -p` or GitHub Actions usage before June 15
4. Read the [quality postmortem](https://www.anthropic.com/engineering) — it's genuinely informative about how product-layer changes can silently degrade AI tool quality
