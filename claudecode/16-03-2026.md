# Claude Code Weekly — March 10–16, 2026

**Busy week.** Four CLI releases, a major new multi-agent Code Review feature hitting GA for teams, and a usage promotion that applies to Claude Code. Here's what matters.

---

## 🚀 What's New

### Claude Code Review — Multi-Agent PR Analysis (Research Preview)
- **What:** Anthropic launched Code Review on March 10, a multi-agent system that automatically deploys specialized AI agents to analyze GitHub pull requests in parallel. Agents scan for logic errors, boundary conditions, and API misuse, then cross-verify each other's findings to reduce false positives. Results post as a summary comment + inline annotations on the PR. Agents **do not** approve PRs — humans still own that.
- **Why it matters:** This is Anthropic's answer to the "who reviews the AI-generated code?" problem. Internal testing showed substantive review comments jumping from 16% → 54% of PRs, with <1% of findings marked incorrect. Large PRs (>1K lines) average 7.5 flagged issues. It's the most ambitious agentic feature Claude Code has shipped for team workflows.
- **Availability:** Team and Enterprise plans only. Admins enable via org settings + GitHub app install. Token-based pricing, averaging **$15–25 per review**. Monthly spend caps available.
- **Source:** [TechCrunch](https://techcrunch.com/2026/03/09/anthropic-launches-code-review-tool-to-check-flood-of-ai-generated-code/), [HelpNetSecurity](https://www.helpnetsecurity.com/2026/03/10/anthropic-claude-code-review/), [MLQ.ai](https://mlq.ai/news/anthropic-launches-code-review-feature-for-claude-code/)

### v2.1.76 "Piebald" — MCP Elicitation Support (March 14)
- **What:** MCP servers can now request structured input from the user mid-task via an interactive dialog with form fields. Previously, MCP tools could only receive parameters at invocation time — now they can pause and ask follow-up questions.
- **Why it matters:** This unblocks a whole class of MCP integrations that need human-in-the-loop decisions during execution (e.g., selecting a deployment target, confirming a DB migration, picking from ambiguous options). If you're building custom MCP servers, this is the feature you've been waiting for.
- **Also in this release:** Context command suggestions, auto memory directory, memory leak fix, permission/auth fixes.
- **Source:** [Claude Code Changelog](https://code.claude.com/docs/en/changelog), [GitHub Releases](https://github.com/anthropics/claude-code/releases), [Releasebot](https://releasebot.io/updates/anthropic/claude)

---

### v2.1.74 — `modelOverrides` for Custom Providers (March 12)
- **What:** New `modelOverrides` setting lets you map model picker entries to custom provider model IDs — including AWS Bedrock inference profile ARNs.
- **Why it matters:** Enterprise teams running Claude through Bedrock or other custom endpoints no longer need workarounds. You can now point the model selector at your own provisioned throughput profiles directly.
- **Also in this release:** Actionable error messages when OAuth/connectivity fails due to SSL cert issues (finally useful guidance for corporate proxy setups with `NODE_EXTRA_CA_CERTS`). Fixed freezes and 100% CPU loops triggered by permission prompts on complex bash commands.
- **Source:** [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

---

### v2.1.72 — `/plan` Gets Arguments, Prompt Caching Fix (March 10)
- **What:** `/plan` now accepts inline arguments (e.g., `/plan fix the auth bug`) to jump straight into plan mode with context. New `/copy` command writes files directly without clipboard. `ExitWorktree` provides clean exits from isolated sessions. Reasoning level indicators simplified to ○ ◐ ●.
- **Why it matters:** The `/plan` change is a small but real workflow speedup — no more typing `/plan` then re-explaining what you want. The prompt caching fix in SDK calls delivers **up to 12× token reduction**, which directly impacts cost and speed for SDK-based integrations. Bundle size also dropped ~510 KB.
- **Other fixes:** `CLAUDE_CODE_DISABLE_CRON` env var to disable in-session cron for `/loop`. Voice mode latency and false "No speech detected" errors fixed.
- **Source:** [Claude Code Changelog](https://code.claude.com/docs/en/changelog), [GitHub Releases](https://github.com/anthropics/claude-code/releases), [jls42.org](https://jls42.org/en/news/ia-actualites-10-mar-2026)

### March Usage Promotion — 2× Off-Peak Limits (Applies to Claude Code)
- **What:** From March 13–27, five-hour usage limits are doubled during off-peak hours (outside 8 AM–2 PM ET on weekdays). Applies to Free, Pro, Max, and Team plans. Bonus usage does **not** count against weekly limits.
- **Why it matters for CLI users:** If you're running long agentic sessions or batch operations via Claude Code, evening/weekend/early-morning work just got significantly more headroom. No action needed — it's automatic.
- **Source:** [Claude Help Center](https://support.claude.com/en/articles/14063676-claude-march-2026-usage-promotion), [XDA](https://www.xda-developers.com/claude-doubled-every-users-usage-limits-for-two-weeks/)

## 🔧 Tips & Tricks

### `/claude-api` Skill for Building on the Anthropic SDK
The new built-in `/claude-api` skill (shipped in v2.1.72) gives you inline reference docs, API guidance, and code patterns for the Claude API and Anthropic SDK without leaving your session. Useful when you're building tools *on top of* Claude while *using* Claude. — [GitHub Releases](https://github.com/anthropics/claude-code/releases), [YouTube](https://www.youtube.com/watch?v=WtMTUSpl1Fc)

### Conductor + Claude Code for Issue-Driven Workflows
Community recommendation from Dev.to: use **Conductor** to orchestrate Claude Code with GitHub/Linear integration. It pulls issue context automatically and wires into CI/CD via GitHub Actions, giving you issue-based agentic development without manual context-stuffing. — [Dev.to](https://dev.to/chand1012/the-best-way-to-do-agentic-development-in-2026-14mn)

### Plans Auto-Save to `.claude/plans/`
When you use plan mode, plans are written to `.claude/plans/{session-slug}.md`. You can review, version-control, or share these. Pro tip from the community: after approving a plan, use the auto-clear context option for better adherence on long tasks. — [Blake Crosley's Guide](https://blakecrosley.com/guides/claude-code)

### Corporate Proxy Users: Check v2.1.74
If you've been fighting SSL errors behind a corporate proxy, the new release finally gives actionable guidance pointing you to `NODE_EXTRA_CA_CERTS`. Previously this just failed silently or with unhelpful messages. — [Claude Code Changelog](https://code.claude.com/docs/en/changelog)
