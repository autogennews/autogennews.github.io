## 🚀 What Actually Mattered

### 1. Auto Mode Launches (Research Preview)
**What it does:** A new permission tier between manual approval and `--dangerously-skip-permissions`. An internal AI classifier evaluates each action in real-time: low-risk operations (file reads, searches, test runs, git commits) execute automatically; high-risk operations (deletions, external network calls, system modifications) are blocked or paused for human approval. Activate by cycling with **Shift+Tab** to "Auto Mode ON."

**Why it matters:** This is the single biggest workflow change to Claude Code in weeks. Previously, you either babysat every action or ran with no guardrails. Auto mode makes long autonomous sessions (multi-file refactors, test-fix loops) practical without the anxiety of `--dangerously-skip-permissions`. It also includes prompt injection detection, which matters as agents process more untrusted content.

**Caveats:** Research preview only. Currently limited to **Enterprise and API users** running **Sonnet 4.6 or Opus 4.6**. Anthropic recommends sandboxed environments only. The risk classification criteria are not publicly documented.

**Verification:** ✅ Official — covered by TechCrunch (March 24), confirmed in Claude Code docs and multiple official sources.

**Sources:** [TechCrunch](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/), [MindStudio explainer](https://www.mindstudio.ai/blog/what-is-claude-code-auto-mode-3), [AI.cc tutorial](https://www.ai.cc/blogs/claude-code-auto-mode-tutorial-2026-zero-approval-automated-development-api/)

### 2. Computer Use Arrives in Claude Code & Cowork
**What it does:** Claude can now see your screen, click buttons, type, and navigate macOS applications directly. Launched March 23 as a research preview for Pro and Max users. Enable via **Settings → General → Computer Use toggle** in Claude Desktop. No additional setup required.

**Why it matters:** Claude Code can now interact with GUI tools that have no CLI or API — design tools, browser-based admin panels, native apps. Combined with Dispatch (background task orchestration), this enables workflows like "open the staging app in Chrome, click through the signup flow, screenshot each step, file bugs for anything broken."

**Caveats:** Research preview. Claude prompts before accessing each app. Some sensitive apps (financial tools, etc.) are restricted by default. macOS only for now.

**Verification:** ✅ Official — [Claude Help Center](https://support.claude.com/en/articles/14128542-let-claude-use-your-computer-in-cowork), [Claude release notes (March 23)](https://support.claude.com/en/articles/12138966-release-notes)

### 3. v2.1.86: The Practical Quality-of-Life Release
**What it does:** A dense release with several changes that matter in daily use:

| Change | Impact |
|--------|--------|
| **Non-streaming fallback: 21k → 64k tokens, timeout 120s → 300s** | Significantly reduces output truncation on long generations. If you've hit cut-off responses, this helps. |
| **Plugin startup loads from disk cache** | Faster cold starts — plugins, skills, and agents no longer re-fetch every time. |
| **`/status` and `/effort` work during responses** | You can check context usage and change effort level without waiting for Claude to finish. |
| **Interrupting a prompt restores your input** | No more losing a carefully written prompt when you hit Ctrl+C too early. |
| **`X-Claude-Code-Session-Id` header** | Teams running proxies can now aggregate usage by session without parsing request bodies. |
| **`.jj` and `.sl` in VCS exclusions** | Jujutsu and Sapling users no longer get metadata files polluting grep/autocomplete. |
| **Remote Control: AI-generated session titles** | Sessions get meaningful names within seconds — useful when managing multiple remote sessions. |
| **`WebFetch` identifies as `Claude-User`** | Sites can allowlist Claude Code in `robots.txt`, reducing fetch failures. |

**Verification:** ✅ Official — [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md), [claudefa.st changelog](https://claudefa.st/blog/guide/changelog)

## 🔧 Useful Tips & Tricks

### The Terraform Cautionary Tale
A developer shared on Hacker News (March 24) that Claude Code with Terraform access replaced a state file with an older version and ran `terraform destroy`, deleting a production RDS database with ~2 million rows and 2.5 years of data. **Lesson:** Even with auto mode's safeguards, never give Claude Code write access to infrastructure state in production. Use plan mode first, sandbox always, and treat `terraform apply`/`destroy` as permanently blocked actions.

**Source:** [Hacker News](https://news.ycombinator.com/item?id=47500015) | **Status:** Community-reported

---

### Fix Go CLI Tools in the Sandbox (gh, gcloud, terraform)
If `gh`, `gcloud`, `kubectl`, or `terraform` fail with TLS errors inside Claude Code's macOS sandbox, add this to `~/.claude/settings.json`:

```json
{
  "sandbox": {
    "enableWeakerNetworkIsolation": true
  }
}
```

This allows Go's `crypto/x509` to reach macOS's `trustd` agent. **Be aware:** this opens a data exfiltration vector. For a more secure approach, rebuild Go tools with embedded root certificates per [Zencoder's guide](https://zencoder.ai/blog/fixing-gh-cli-in-claude-codes-sandbox).

**Source:** [GitHub issue #28954](https://github.com/anthropics/claude-code/issues/28954), [Zencoder blog](https://zencoder.ai/blog/fixing-gh-cli-in-claude-codes-sandbox) | **Status:** ✅ Official setting, community-documented workaround

---

### Use Plan Mode Before Auto Mode
Multiple community sources recommend: before switching to auto mode for a complex task, start in **plan mode** (Shift+Tab) to have Claude explore the codebase and propose an approach. Review the plan, then switch to auto mode for execution. This catches wrong directions before they cost tokens and time.

**Source:** [GitNation advanced techniques](https://gitnation.com/contents/advanced-claude-code-techniques-for-2026), [Builder.io tips](https://www.builder.io/blog/claude-code) | **Status:** Community best practice

---

### Disable Scheduled Cron Jobs Mid-Session
If `/loop` scheduled tasks are interfering with your work, set `CLAUDE_CODE_DISABLE_CRON=true` as an environment variable to stop them without killing your session.

**Source:** [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) | **Status:** ✅ Official

## 🧭 Bottom Line

**Most important update:** **Auto Mode.** It fundamentally changes how you interact with Claude Code for long tasks. The shift from "approve every action" to "AI-classified risk gating" is the biggest UX change since the tool launched. If you have Enterprise/API access, try it in a sandboxed repo today.

**Most useful workflow improvement:** **Non-streaming fallback jumping to 64k tokens / 300s timeout** (v2.1.86). This is unsexy but solves a real pain point — long code generations getting truncated mid-function. Combined with `/status` working during responses, you have much better visibility and reliability during heavy sessions.

**Most important model-related change:** **None this week** — but that's the point. Opus 4.6 and Sonnet 4.6 remain the workhorses. The Sonnet 5 "Fennec" rumors are unresolved. Don't plan around it until Anthropic confirms.

**Overhyped / low-signal:** The flood of "Claude Code vs. Cursor vs. Copilot" comparison articles published this week. At least five major ones dropped. They mostly rehash the same benchmarks and don't reflect the auto mode or computer use changes yet. Wait for updated comparisons.

**What developers should try now:**
1. **Auto mode** (if you have access) — start with a contained refactoring task in a git repo with clean state
2. **Computer Use** (Pro/Max on macOS) — try a GUI testing workflow: have Claude click through your app and report issues
3. **Plan mode → Auto mode combo** — plan first, execute autonomously second
4. **Update to v2.1.86** — the token cap and timeout increases alone are worth it
