### **`/loop` — Cron-scheduled recurring tasks (v2.1.71, Mar 8)**
The marquee feature of v2.1.71. You can now schedule recurring prompts inside an active Claude Code session using natural-language intervals.

```
/loop every 15 minutes scan logs for new errors and summarize
/loop 5m check the deploy
/loop babysit all my PRs
```

Minimum interval is 1 minute. Loops are **session-scoped** (they stop when you close the session), include a ~10% random offset to avoid API spikes, and can run for up to 3 days. For longer scheduling, Anthropic points you to Claude Code Desktop tasks or GitHub Actions.

**Why it matters:** This turns Claude Code into a lightweight monitoring agent. PR babysitting, deployment health checks, log scanning — tasks you'd normally set up a separate cron job or CI step for can now live in your terminal session. Particularly powerful when combined with MCP servers (e.g., Slack, Firecrawl).

**Source:** [jls42.org news roundup](https://jls42.org/en/news/ia-actualites-8-mar-2026) · [mejba.me deep dive](https://www.mejba.me/blog/claude-code-loop-cron-scheduling) · [YouTube walkthrough](https://www.youtube.com/watch?v=pWZh37iRnDA)

### **Native Git Worktree Support (v2.1.49, Feb 19)**
Claude Code now has a built-in `--worktree` (`-w`) flag that creates an isolated git worktree, branches off your default remote, and runs the agent inside it.

```bash
claude -w auth-feature    # creates .claude/worktrees/auth-feature/
claude --worktree          # auto-generates a name
```

Worktrees share the `.git` database (no full clone), auto-clean if no changes are made, and prompt before deletion if changes exist. The Desktop app has a toggle for automatic worktree isolation per session.

**Why it matters:** Running multiple Claude Code agents in parallel on the same repo was always possible but required manual worktree setup. This removes that friction entirely. Spin up 3 agents on 3 features, merge when done.

**⚠️ Known issue:** The `-w` flag is [silently ignored in some environments](https://github.com/anthropics/claude-code/issues/27315) — test with `--debug` if it doesn't seem to work.

**Source:** [Substack announcement](https://open.substack.com/pub/aidisruption/p/claude-code-adds-native-git-worktree) · [Verdent setup guide](https://www.verdent.ai/es/guides/claude-code-worktree-setup-guide) · [Stitcher.io](https://stitcher.io/feed/posts/74305)

### **Claude Code Security — AI vulnerability scanning (Feb 20)**
Anthropic launched **Claude Code Security**, an AI-powered codebase scanner powered by Opus 4.6. It goes beyond pattern-matching static analysis — it traces data flows, analyzes component interactions, and detects memory corruption, injection flaws, auth bypasses, and logic errors. Multi-stage verification reduces false positives, and it suggests patches that require human approval.

**Availability:** Limited research preview for **Enterprise and Team** customers, with expedited access for open-source maintainers.

**Why it matters:** Anthropic claims 500+ previously unknown high-severity vulnerabilities found in production open-source codebases during internal testing. If the false-positive rate is genuinely low, this could replace or supplement tools like Snyk/Semgrep for deep logic-level bugs that traditional SAST misses.

## 🔧 Tips & Tricks

**Tiered thinking modes for cost/quality control**
Use `think` for straightforward tasks, `think hard` for edge cases and performance optimization, and `ultra think` for complex architectural decisions. Each level increases reasoning depth (and token usage). Match the mode to the task to avoid burning tokens on simple fixes.
**Source:** [claude-world.com guide](https://claude-world.com/articles/claude-code-complete-guide-2026)

**Skills files for team-standardized prompts**
Drop `.md` files in `~/.claude/skills/` (global) or `.claude/skills/` (project) to create reusable slash commands like `/pr`, `/review`, `/migrate`. Each file's content becomes the prompt template. Combine with hooks for enforcement — e.g., a PreToolUse hook that runs `prettier` before any file write.
**Source:** [okhlopkov.com real-world setup](https://okhlopkov.com/claude-code-setup-mcp-hooks-skills-2026/)

**Four hook types — pick the right one**
The hooks system now supports `command` (shell — fast, free), `http` (POST to URL — for external integrations), `prompt` (single-turn LLM evaluation — for subjective checks), and `agent` (spawns a sub-agent with tools — for deep verification). Start with `command` for speed; escalate to `prompt`/`agent` only when you need judgment, not just a binary check.
**Source:** [Dev.to hooks guide](https://dev.to/serenitiesai/claude-code-hooks-guide-2026-automate-your-ai-coding-workflow-dde) · [wmedia.es practical guide](https://wmedia.es/en/writing/claude-code-hooks-practical-guide)

**New environment variables for enterprise scripting**
v2.1.69+ exposes `CLAUDE_CODE_ACCOUNT_UUID`, `CLAUDE_CODE_USER_EMAIL`, and `CLAUDE_CODE_ORGANIZATION_UUID` — useful for audit logging, per-user hook behavior, or conditional configs in shared CI environments.
**Source:** [seanpan.me AI landscape roundup](https://seanpan.me/ai/claude0326/)

**Plan mode first, then execute**
Community consensus: start sessions with a planning prompt ("How would you approach X?"), review the generated plan, then approve execution. This catches architectural mistakes before Claude starts writing code. Pair Opus-tier models for planning with Sonnet for implementation to balance cost and quality.
**Source:** [faros.ai model comparison](https://www.faros.ai/blog/best-ai-model-for-coding-2026) · [hackceleration.com review](https://hackceleration.com/claude-code-review/)
