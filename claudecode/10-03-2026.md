# Claude Code Developer Briefing — Feb 9 – Mar 10, 2026

This was a packed period. Three point releases landed in the first week of March, two new model versions shipped in February, a serious security disclosure dropped, and the community ecosystem around hooks/MCP/skills matured significantly. Here's what matters.

## 🚨 Security Advisory (Read This First)

**CVE-2025-59536 & CVE-2026-21852 — RCE and API Key Exfiltration via Malicious Repos**

On February 25, Check Point Research publicly disclosed two vulnerabilities in Claude Code's project-load flow. The attack vector: a malicious repository could include crafted Hooks configs, MCP server definitions, or environment variable overrides that execute arbitrary shell commands or redirect your API key to an attacker-controlled endpoint — **before the trust prompt even appeared**.

- **CVE-2026-21852** (CVSS 5.3): A repo-level settings file could set `ANTHROPIC_BASE_URL` to an attacker's server. Claude Code would send your plaintext API key on first request, no user interaction required.
- **CVE-2025-59536**: Broader RCE chain exploiting Hooks and MCP configs to achieve reverse shells on clone-and-open.

**Patched in version 2.0.65** (January 2026). If you have auto-update enabled, you're already covered. If not: **update immediately**. And as a general rule, treat `.claude/` directories in untrusted repos with the same suspicion you'd give a `Makefile` or `postinstall` script.

**Sources:** [Check Point Research](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/) · [The Hacker News](https://thehackernews.com/2026/02/claude-code-flaws-allow-remote-code.html) · [The Register](https://www.theregister.com/2026/02/26/clade_code_cves/) · [NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-21852)

### Model Updates

**Claude Opus 4.6 — released Feb 5, 2026**
Anthropic's new flagship model, now available in Claude Code. Opus 4.6 includes a fast mode with the **full 1M-token context window**, meaning you can feed massive codebases into a single session without truncation. It also introduced agent team functionality. ### February Platform Features (from mid-Feb releases)

Based on aggregated reporting from the period, these features also shipped in February:

**HTTP Hooks**
Hooks can now make POST requests to external URLs with JSON payloads and receive responses — enabling external service integration (Slack notifications, Datadog metrics, webhook triggers) without shelling out. Config example:
```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{
        "type": "http",
        "url": "https://your-service.com/webhook",
        "timeout": 30
      }]
    }]
  }
}
```
**Why it matters:** You can now wire Claude Code into your team's observability and notification stack declaratively.

**Worktree Config Sharing**
Project configurations and auto-memory are now shared across git worktrees of the same repository. If you use `git worktree` for parallel feature branches, Claude Code no longer treats each worktree as a separate project.

**Managed Settings (Enterprise)**
Support for enterprise-managed configuration via macOS plist or Windows Registry. IT teams can now enforce Claude Code policies (allowed models, permitted tools, hook restrictions) at the device level.

**New Environment Variables:**
- `CLAUDE_CODE_ACCOUNT_UUID`
- `CLAUDE_CODE_USER_EMAIL`
- `CLAUDE_CODE_ORGANIZATION_UUID`

Useful for hooks and integrations that need to identify the current user/org context.

**Data Residency Controls**
New `inference_geo` parameter enables US-only inference at 1.1× pricing for models released after Feb 1, 2026. Relevant for teams with compliance requirements.

## 🔧 Tips & Tricks

**Use Hooks + MCP as a governance layer, not just automation**
The emerging pattern is: MCP servers provide *capabilities* (fetch from GitHub, query a DB), while Hooks provide *lifecycle control* (audit before execution, notify after). A `PreToolUse` hook can enforce policies like "block production DB writes without approval"; a `PostToolUse` hook can send a Slack message after every PR creation. This combo is becoming the standard for teams that want agentic power with guardrails.
**Source:** [Tech Bytes — Mastering Hooks](https://techbytes.app/posts/mastering-claude-code-hooks-2026/) · [Oflight — MCP Integration Guide](https://www.oflight.co.jp/en/columns/claude-code-mcp-integration-guide-2026)

**GSD (Get Shit Done) — a community workflow framework worth knowing**
GSD is a spec-driven development framework (23k GitHub stars) built entirely on Claude Code slash commands. The core idea: `/gsd:new-project` generates structured planning files (requirements, roadmap, state), then `/gsd:plan-phase N` breaks each phase into tasks. Each phase runs in a fresh context window to avoid drift, and every step auto-commits to git. It's opinionated but solves the real problem of context degradation in long sessions.
**Source:** [codecentric deep dive](https://www.codecentric.de/en/knowledge-hub/blog/the-anatomy-of-claude-code-workflows-turning-slash-commands-into-an-ai-development-system)

**Boris Cherny's workflow: parallel instances + custom slash commands for every inner-loop task**
Claude Code's creator shared his daily workflow in multiple interviews this period. Key takeaways: he runs parallel Claude Code instances for independent tasks, uses custom slash commands for every repeated action (reviews, migrations, test fixes), and structures PRs to be small and deterministic. His philosophy: "slash commands are how you encode your team's engineering standards into the tool."
**Source:** [Lenny's Newsletter](https://www.lennysnewsletter.com/p/head-of-claude-code-what-happens) · [InfoQ](https://www.infoq.com/news/2026/01/claude-code-creator-workflow/) · [Pragmatic Engineer interview (YouTube)](https://www.youtube.com/watch?v=julbw1JuAz0)

**Use Opus for planning, Sonnet for execution**
Multiple community sources converge on this pattern: use Opus 4.6 (or `/plan` mode) for architecture decisions and complex reasoning, then switch to Sonnet 4.6 for implementation. The cost difference is significant and Sonnet 4.6 is fast enough that the context-switch overhead is minimal.
**Source:** [Faros — Best AI Models for Coding](https://www.faros.ai/blog/best-ai-model-for-coding-2026)

**Post-write formatting hook — auto-run Prettier/ESLint after every file write**
A simple but high-value hook pattern:
```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{
        "type": "command",
        "command": "prettier --write \"$CLAUDE_FILE_PATH\""
      }]
    }]
  }
}
```
Ensures every file Claude touches matches your project's formatting rules without manual intervention.
**Source:** [yuv.ai — Hooks Mastery](https://yuv.ai/blog/claude-code-hooks-mastery)
