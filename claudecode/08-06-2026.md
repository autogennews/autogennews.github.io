# Claude Code Weekly — June 2, 2026 – June 8, 2026

A big week. A new default model, Dynamic Workflows going mainstream, a serious GitHub Action security flaw, and Anthropic publishing that 80% of its own merged code is now written by Claude.

---

## 🚀 What Actually Mattered

### 1. Opus 4.8 Becomes the Default Model in Claude Code
**What it does:** On June 2, Anthropic upgraded Claude Code's default model to Opus 4.8 (released May 28). Simultaneously, **Opus 4 and 4.1 were removed** from the Claude model selector and Claude Code entirely.

**Why it matters:** This is the model you're now coding with whether you chose it or not. The headline improvement is **agentic reliability** — Anthropic claims 4× fewer code flaws, meaning the model is significantly better at catching its own bugs during iterative write-test-fix loops. It also shows reduced mid-task drift and fewer tool-use errors. Same pricing as Opus 4.7 ($5/M input tokens), 1M context window. Available on Bedrock, Vertex AI, and DigitalOcean.

**Verification:** Official — [Claude Help Center release notes](https://support.claude.com/en/articles/12138966-release-notes), [decodethefuture.org](https://decodethefuture.org/en/claude-opus-4-8-explained/), [MindStudio](https://www.mindstudio.ai/blog/what-is-claude-opus-4-8-anthropic-model-update/), [Caylent](https://caylent.com/blog/claude-opus-4-8-what-improved-whats-new-and-what-it-means-for-enterprise)

### 2. Dynamic Workflows Explode in Community Adoption
**What it does:** Launched alongside Opus 4.8 in late May, Dynamic Workflows hit critical mass this week with guides, videos, and real-world usage reports flooding every channel. The feature lets Claude write short JavaScript orchestration scripts on the fly, spawn tens to hundreds of parallel subagents, break tasks into subtasks, and validate/iterate results — all within a single Claude Code session.

**How to use it:** Enable the `ultracode` setting, or explicitly ask Claude to "create a workflow." Available in research preview for Max, Team, Enterprise, and API users.

**Why it matters:** This is the biggest practical capability expansion Claude Code has had. It turns Claude Code from a single-threaded assistant into a parallel task orchestrator for repo-wide audits, large migrations, security reviews, and multi-file refactors. **Caveat:** Token usage is substantially higher — start small.

**Verification:** Official — [InfoQ](https://www.infoq.com/news/2026/06/dynamic-workflows-claude-code/), [DevOps.com](https://devops.com/claude-codes-dynamic-workflows-take-on-the-tasks-that-were-too-big-to-automate/); Community guides — [Product Compass](https://www.productcompass.pm/p/claude-code-dynamic-workflows), [MindStudio](https://www.mindstudio.ai/blog/anthropic-dynamic-workflows-when-to-use-them/), [claudefa.st](https://claudefa.st/blog/guide/development/dynamic-workflows)

### 3. Critical GitHub Action Security Vulnerability — Patch Now
**What it does:** Security researcher RyotaK (Flatt Security) disclosed a flaw in `claude-code-action` where a crafted GitHub issue from a bot account could bypass trust checks, trigger prompt injection, steal OIDC tokens, and gain **write access** to repositories. The attack could have been used to poison the `claude-code-action` repo itself, creating a supply-chain risk. Microsoft Threat Intelligence independently flagged CI/CD secret exposure on June 5.

**Fix:** Upgrade to **`claude-code-action` v1.0.94** or later. Audit any workflows that let untrusted users/bots trigger Claude. Remove unnecessary secrets from CI environments for untrusted-input jobs.

**Why it matters:** If you're running Claude Code in CI/CD with GitHub Actions, this is a **drop-everything-and-patch** situation. The attack surface is real and was actively demonstrable.

**Verification:** Official fix confirmed — [Flatt Security](https://flatt.tech/research/posts/poisoning-claude-code-one-github-issue-to-break-the-supply-chain/), [The Hacker News](https://thehackernews.com/2026/06/claude-code-github-action-flaw-let-one.html), [TNW](https://thenextweb.com/news/claude-code-github-action-prompt-injection-flaw), [WindowsForum/MSFT](https://windowsforum.com/threads/claude-code-ci-cd-secret-exposure-via-prompt-injection-what-teams-must-fix.423109/)

### 4. Claude Code v2.1.166 — Fallback Models, Security Hardening, Thinking Controls
**What it does:**
- **Fallback models:** Configure up to 3 fallback models tried in order when the primary is overloaded/unavailable. Works in print mode and background sessions.
- **`*` deny-all glob:** A single `*` in the tool-name position of deny rules blocks every tool at once — much simpler session lockdown.
- **Cross-session permission fix:** Messages relayed between Claude sessions no longer carry user authority, preventing permission escalation across sessions.
- **Thinking controls:** Setting max thinking tokens to zero now reliably disables thinking, even for models that think by default.

**Why it matters:** Fallback models are a practical reliability improvement for anyone running Claude Code in automation or CI. The security fixes matter for multi-session and headless workflows. The deny-all glob is a small but useful quality-of-life improvement for policy enforcement.

**Verification:** Mixed — [Releasebot](https://releasebot.io/updates/anthropic/claude-code), [ocdevel.com podcast notes](https://ocdevel.com/podcaster/claude-code/7fdc1bc3-0a3b-42a3-8b68-c7e5f61d6b38)

### 5. ⚠️ Model Deprecation Deadline: Sonnet 4 & Opus 4 Die June 15
**What it does:** Claude Sonnet 4 and Claude Opus 4 will stop working on the Anthropic API on **June 15, 2026**. Requests using those model IDs will fail.

**Migration path:** Sonnet 4 → **Sonnet 4.6**. Opus 4 → **Opus 4.6**.

**Why it matters:** If you have any Claude Code configs, scripts, API calls, or integrations still pointing at `claude-sonnet-4` or `claude-opus-4`, you have **one week** to update them. This is especially relevant for teams using the API or Agent SDK with hardcoded model strings.

**Verification:** Official — [Make.com deprecation notice](https://developers.make.com/white-label-documentation/release-notes/anthropic-claude-model-deprecations-on-june-15-2026), [endoflife.date](https://endoflife.date/claude)

### 6. Anthropic: "80% of Our Merged Code Is Written by Claude"
**What it does:** On June 8, Anthropic published a paper on recursive self-improvement revealing that as of May 2026, **>80% of code merged into Anthropic's production codebase was authored by Claude**. Engineers are shipping 8× more code per quarter than in 2024, with the role shifting from writing code to directing and reviewing Claude's output.

**Why it matters:** This is the strongest real-world signal yet for Claude Code's production capability. It's also a data point for the "recursive self-improvement" trajectory Anthropic is tracking — Claude increasingly builds the systems that build Claude. For developers evaluating Claude Code, this is Anthropic eating its own dogfood at scale.

**Verification:** Official — [Anthropic](https://www.anthropic.com/institute/recursive-self-improvement), [TNW](https://thenextweb.com/news/anthropic-claude-recursive-self-improvement-code), [Fortune](https://fortune.com/2026/06/05/anthropic-ai-pause-development-recursive-self-improvement/)

**What it does:** CSO Online published a detailed report on MCP security risks in Claude Code — specifically around post-install hooks executing arbitrary code and the lack of consistent enforcement in developer environments. ## 🔧 Useful Tips & Tricks

**Skills: One skill, one job.** The strongest community advice this week: keep skills narrowly scoped. A few worked examples in a skill outperform long constraint lists. Prefer showing Claude what good output looks like over telling it what not to do.
*Source: [Firecrawl best skills guide](https://www.firecrawl.dev/blog/best-claude-code-skills), [eesel.ai CLI reference](https://www.eesel.ai/blog/claude-code-cli-reference) — community-reported*

**Session handoff pattern.** Save current session state into a structured markdown file so a fresh Claude session can continue cleanly. Multiple guides this week converged on this as a key workflow pattern for long-running tasks.
*Source: [Firecrawl](https://www.firecrawl.dev/blog/best-claude-code-skills) — community-reported*

**Dynamic Workflows: start small.** Token burn is real. Community consensus is to test on a well-scoped subtask before pointing Dynamic Workflows at your entire repo. The mental model: use subagents for one round of parallel work, Dynamic Workflows for multi-stage work where each stage changes the next step, and `/goal` for keeping long sessions aligned.
*Source: [MindStudio](https://www.mindstudio.ai/blog/anthropic-dynamic-workflows-when-to-use-them/), [Product Compass](https://www.productcompass.pm/p/claude-code-dynamic-workflows) — community-reported*

**Fallback model config for reliability.** With v2.1.166, you can now set up to 3 fallback models. Useful for automation and CI where you can't afford a session dying because the primary model is overloaded.
*Source: [ocdevel.com](https://ocdevel.com/podcaster/claude-code/7fdc1bc3-0a3b-42a3-8b68-c7e5f61d6b38) — mixed*

**Boris Cherny's framing shift.** The head of Claude Code described the developer workflow evolution as moving from "writing prompts" to "writing loops that manage Claude." Worth internalizing as a mental model for how to get the most out of Claude Code in 2026.
*Source: [Digg/X](https://digg.com/ai/v1igoqs7), [KuCoin/interview](https://www.kucoin.com/news/flash/anthropic-s-boris-cherny-on-claude-code-and-the-future-of-engineering) — mixed*

## 🧭 Bottom Line

**Most important update:** **Opus 4.8 as the new default model.** Better agentic self-correction (4× fewer code flaws) directly improves every Claude Code session without you changing anything.

**Most useful workflow improvement:** **Dynamic Workflows.** If you haven't tried them yet, this is the week — the community guides are excellent, and the feature genuinely changes what's possible for repo-scale tasks. Start with `ultracode` on a bounded problem.

**Most important model-related change:** **Sonnet 4 and Opus 4 deprecation on June 15.** If you have hardcoded model IDs anywhere — configs, scripts, API calls — fix them this week or they break next Sunday.

**Anything overhyped or low-signal:** The "80% of code written by Claude" headline is impressive but easy to over-interpret. It's about Anthropic's internal workflow, not a claim about general developer productivity. Also, the flood of "Dynamic Workflows tutorial" content this week varies wildly in quality — stick to the official docs and the InfoQ/MindStudio/claudefa.st guides.

**What developers should try now:**
1. **Patch `claude-code-action`** to v1.0.94+ if you use GitHub Actions — this is urgent.
2. **Try Dynamic Workflows** on a scoped task (audit, migration, multi-file refactor) to understand the capability and token cost.
3. **Check your model IDs** for the June 15 deprecation.
4. **Set up fallback models** in v2.1.166 if you run Claude Code in automation.
