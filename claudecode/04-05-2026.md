## 🚀 What Actually Mattered

### 1. Claude Code v2.1.126: The Hardening Release (May 1)

**What it does:** A 33-change release focused on security, platform reliability, and quality-of-life improvements. The headline features:

- **`claude project purge [path]`** — New command that deletes all Claude Code state for a project (transcripts, tasks, file history, config) in one shot. Supports `--dry-run`, `--interactive`, and `--all` flags. This matters because Claude Code accumulates a chronological log of every prompt you've sent — now you can actually clean that up.
- **Gateway-aware model picker** — `/model` now queries your gateway's `/v1/models` endpoint when `ANTHROPIC_BASE_URL` points to an Anthropic-compatible proxy. Useful for teams running through corporate gateways or load balancers.
- **OAuth over SSH, WSL, and containers** — `claude auth login` now accepts pasted OAuth codes when browser callbacks can't reach localhost. This was a major pain point for remote dev setups.
- **Mac sleep survival** — Sessions no longer crash when your Mac sleeps.
- **Clipboard security fix** — Clipboard writes no longer expose copied content as command-line arguments visible to EDR/SIEM telemetry tools.
- **CJK rendering fix** — Japanese, Korean, and Chinese text no longer garbles on Windows in no-flicker mode.
- **Permission check stall indicator** — Red spinner now shows when permission checks hang, instead of silently appearing to run.

**Why it matters:** This is a stability and trust release. The `project purge` command and clipboard fix address real security/privacy gaps. The OAuth fix unblocks a large class of remote developers. None of these are flashy, but they reduce daily friction.

**Verification:** Official — confirmed via [Releasebot](https://releasebot.io/updates/anthropic/claude-code), [WotAI](https://wotai.co/blog/claude-code-2-1-126), [FindSkill](https://findskill.ai/blog/claude-code-2-1-126-project-purge-cleanup/)

### 2. 1M Token Context Beta Retired for Sonnet 4.5 & Sonnet 4 (April 30)

**What it does:** The `anthropic-beta: context-1m-2025-08-07` header no longer works on Sonnet 4.5 or Sonnet 4. Requests exceeding 200K tokens on these models now return errors.

**Why it matters:** **Breaking change** for anyone still using older Sonnet models with extended context via the API (including custom Claude Code configurations pointing at these models). The fix is a 2-line migration: drop the beta header, switch to `claude-sonnet-4-6` (which has 1M context at GA, same $3/$15 pricing).

**Verification:** Official — [Releasebot/Anthropic release notes](https://releasebot.io/updates/anthropic), [FindSkill migration guide](https://findskill.ai/blog/migrate-claude-sonnet-4-5-to-4-6/)

### 3. The Postmortem Fallout: Community Recalibrates Effort Levels

**What it does:** Anthropic's April 23 postmortem — explaining three compounding regressions (reasoning effort reduction, caching bug, verbosity prompt change) that degraded Claude Code quality through March–April — dominated developer discussion this week. All issues were reverted by April 20 (v2.1.116), but the community spent this week adjusting workflows in response.

**Why it matters:** The most-discussed HN thread ([#47957402](https://news.ycombinator.com/item?id=47957402)) reframed how developers should think about effort levels: *"Opus 4.7's default is xhigh. I set the level to fit the work. A quick edit doesn't need max, an architectural decision does. The cheapest move and the one I see people skip."* The postmortem also validated that quality is back to expected levels, giving confidence to developers who had been hedging with alternative tools.

## 🔧 Useful Tips & Tricks

### Match effort level to task complexity
From the widely-discussed HN postmortem thread: explicitly set `/effort` per task instead of relying on defaults. Use `high` or `xhigh` for complex refactoring/architecture; drop to `standard` or `low` for quick edits. Opus 4.7 defaults to `xhigh`; other models default to `high`. The latency difference is 2.8–5.3× between `standard` and `xhigh`, so over-specifying wastes time on simple tasks.

**Source:** [HN #47957402](https://news.ycombinator.com/item?id=47957402), [Damian Galarza blog](https://www.damiangalarza.com/posts/2026-04-30-claude-opus-4-7-claude-code-tips-extended-context/) | **Status:** Community-validated, consistent with official docs

### Encode session learnings into CLAUDE.md or skills
Top-voted prompting pattern: end sessions with *"Reflect on this session and encode via claude.md or skills what you learned."* This creates persistent memory across sessions, reducing repeated mistakes. Pair with separate agents for spec review vs. code review to isolate concerns.

**Source:** [HN #47957402](https://news.ycombinator.com/item?id=47957402) | **Status:** Community tip

### Use hooks for non-negotiable automation
Hooks execute deterministically every time, unlike prompt instructions which can be ignored. Push linting, formatting, security scans, and git backups into hooks rather than relying on the model to remember. Example: pre-write hook running `git stash push -m "claude-backup"`.

**Source:** [Blake Crosley guide](https://blakecrosley.com/guides/claude-code), [GitHub best practices repo](https://github.com/shanraisshan/claude-code-best-practice) | **Status:** Community best practice, consistent with official docs

### Start simple: CLAUDE.md + 2–3 skills
Pedro Sant'Anna's widely-shared workflow guide runs 14 agents, 30 skills, and 24 rules — but explicitly says *"That is the ceiling, not the floor. Start with just CLAUDE.md and 2–3 skills."* Over-engineering your Claude Code setup before understanding the primitives is a common mistake.

**Source:** [Sant'Anna workflow guide](https://psantanna.com/claude-code-my-workflow/workflow-guide.html) | **Status:** Community best practice

## 🧭 Bottom Line

**Most important update:** **v2.1.126's OAuth fix for SSH/WSL/containers.** This unblocked a real class of developers who couldn't authenticate in remote environments — a prerequisite for everything else.

**Most useful workflow improvement:** **Matching `/effort` to task complexity.** The postmortem discussion crystallized what many suspected: blindly running at max effort wastes time and tokens, while under-specifying loses quality. Being intentional about this is the highest-ROI habit change this week.

**Most important model-related change:** **1M context beta retirement for Sonnet 4.5/4.** If you're still on these models, this is a breaking change. Migrate to Sonnet 4.6 — it's a 2-line fix and the pricing is the same.

**Overhyped / low-signal:** The Pro plan pricing scare generated outsized anxiety relative to its current impact. Claude Code is still on Pro. Worth monitoring, not worth panicking over.

**What developers should try now:**
1. Update to v2.1.126 (`claude update`)
2. Run `claude project purge --dry-run` to see what's accumulated
3. If you're on Sonnet 4.5 via API, migrate to 4.6 before you hit errors
4. Add an `/effort` habit: `xhigh` for architecture, `standard` for quick fixes
5. Clear your calendar for May 6 — the Code with Claude conference is likely to bring a model drop
