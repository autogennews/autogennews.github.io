## 🚀 What Actually Mattered

### 1. The Source Code Leak (March 31)

**What happened:** A misconfigured Bun build shipped a 59.8 MB source map inside `@anthropic-ai/claude-code` v2.1.88 on npm, exposing ~500,000 lines of readable TypeScript — the entire agentic harness.

**Why it matters:** This wasn't a model weights leak, but it exposed the full architecture of how Claude Code orchestrates tool use, guardrails, permissions, and prompts. More interesting for developers: the code revealed **108 gated feature flags** for unshipped capabilities, including:

- **ULTRAPLAN** — offloads complex task planning to Opus in the cloud for up to 30 min, with browser-based approval before local execution
- **Coordinator Mode** — one Claude instance spawning parallel sub-agents for multi-agent orchestration
- **Undercover Mode** — strips AI attribution from commits and hides internal codenames (active for Anthropic employees on public repos)
- **Companion Pet System** — deterministic ASCII Tamagotchi-style pets seeded by user ID (planned rollout window: April 1–7)
- **Anti-distillation poisoning** — injects fake tool schemas to corrupt competitor training data

The code hit 50,000 GitHub stars in 2 hours and remains mirrored despite DMCA takedowns. Anthropic called it "human error in release packaging." No customer data or credentials were exposed.

**Verification:** Official (confirmed by Anthropic) + extensively covered by Fortune, Axios, TechCrunch, Cybernews, Dev.to

### 2. v2.1.89 — "Defer" for Headless Hooks (April 1)

**What it does:** Adds a third permission decision to PreToolUse hooks: `"defer"`. Previously hooks could only `allow` or `deny` a tool call. Now they can **pause execution** in headless (`-p`) sessions, holding state until you resume with `claude -p --resume` or `-r <session-id>`.

Also shipped:
- **`CwdChanged` and `FileChanged` hook events** — fire when the working directory changes or files are modified, enabling direnv-style automatic environment reloading
- Allow rules now check **resolved symlink targets**, not just requested paths
- Fixes for Edit/Write doubling CRLF on Windows, schema cache bugs (~50% failure rate with multiple schemas), memory leaks on large JSON inputs, autocompact thrash loops, and LSP zombie states

**Why it matters:** `defer` is the missing piece for production CI/CD with Claude Code. You can now build approval gates — pause before destructive operations, wait for human sign-off via Slack/webhook, then resume without losing session state. The `CwdChanged`/`FileChanged` hooks are a quality-of-life win for monorepo and polyglot setups where environment context shifts constantly.

**Verification:** Official
**Sources:** [GitHub release](https://github.com/anthropics/claude-code/releases), [Official changelog](https://code.claude.com/docs/en/changelog), [SourceForge mirror](https://sourceforge.net/projects/claude-code.mirror/files/v2.1.89/)

### 3. v2.1.92 — Enterprise Controls + Bedrock Wizard (April 4)

**What it does:**
- **Bedrock setup wizard** — interactive flow on the login screen that walks through AWS auth, region selection, credential verification, and model pinning. No more manual env var juggling.
- **`forceRemoteSettingsRefresh`** in managed-settings — CLI blocks startup until it fetches the latest remote policy; exits (fail-closed) if fetch fails. Prevents local cache drift.
- **MCP tool result persistence** up to 500K characters
- **`/powerup` command** — interactive tutorial with animated demos for new features
- **Prompt-cache hints** and **faster large-file writes**
- **`sandbox.failIfUnavailable`** setting to enforce sandboxing

Bug fixes that actually matter:
- **Edit/Write "File content has changed"** error when a PostToolUse format-on-save hook rewrites the file between consecutive edits — this was a real pain point for anyone using Prettier/Black/rustfmt hooks
- PreToolUse hooks emitting JSON to stdout with exit code 2 now correctly block tool calls
- Removed `Get-DnsClientCache` and `ipconfig /displaydns` from auto-allow (DNS cache privacy on Windows)

**Why it matters:** The Bedrock wizard removes a significant friction point for AWS shops. `forceRemoteSettingsRefresh` is a real enterprise governance feature — IT can now guarantee policy enforcement at startup. The format-on-save fix unblocks a very common hook pattern.

### 4. OpenClaw Third-Party Tool Pricing Cut-Off (April 4)

**What happened:** As of noon PT on April 4, Claude Pro and Max subscriptions **no longer cover OpenClaw or other third-party harness usage**. These now require separate pay-as-you-go "extra usage" billing.

**Why it matters:** This is a significant cost increase for developers who integrated Claude Code with OpenClaw for extended toolchains. Reports cite potential jumps from ~$20/month to $500+ for heavy autonomous agent usage (100K+ tokens/hour at $3/$15 per million input/output tokens for Sonnet 4.6). Anthropic is offering a one-time credit equal to one month's subscription, redeemable until April 17. Pre-purchase bundles offer up to 30% discount.

**Verification:** Official (Anthropic announcement) + widely covered

### 5. API Platform: 300K Batch Output + 1M Context Retirement for Older Models (March 30)

**What changed:**
- **Message Batches API max_tokens raised to 300K** for Opus 4.6 and Sonnet 4.6 (requires `output-300k-2026-03-24` beta header)
- **1M token context window beta retiring for Sonnet 4.5 and Sonnet 4** on April 30, 2026. The `context-1m-2025-08-07` header will stop working.

**Why it matters for Claude Code users:** If you're running Claude Code against the API on older Sonnet models with 1M context, you need to migrate to Sonnet 4.6 or Opus 4.6 before April 30. The newer models support 1M context natively without a beta header. The 300K batch output is relevant for bulk code generation pipelines.

## 🔧 Useful Tips & Tricks

**Use `defer` hooks for CI/CD approval gates**
Return `{"decision": "defer"}` from a PreToolUse hook to pause headless sessions before destructive operations (e.g., `rm`, `git push --force`). Resume with `claude -p --resume` after external approval. This is the cleanest way to add human-in-the-loop to automated Claude Code pipelines.
*Source: [Official hooks docs](https://code.claude.com/docs/en/hooks), [ClaudeLab guide](https://claudelab.net/en/articles/claude-code/claude-code-april-2026-update-deferred-permission-mcp) — Official*

**`CwdChanged` + direnv for automatic environment loading**
Hook into `CwdChanged` to run `direnv allow && eval "$(direnv export json)"` whenever Claude Code switches directories. Essential for monorepos with per-directory `.envrc` files. Note: hooks may fire twice from the home directory — use file locking for deduplication.
*Source: [ClaudeLab guide](https://claudelab.net/en/articles/claude-code/claude-code-cwdchanged-filechanged-hooks-guide) — Official/community*

**Try `/powerup` for interactive feature discovery**
New in v2.1.90+. Launches animated interactive tutorials for recent features like MCP persistence and plugin directories. Faster than reading changelogs.
*Source: [apiyi.com](https://help.apiyi.com/en/claude-code-v2-1-92-mcp-persistence-powerup-tutorial-en.html) — Official*

**`.claudeignore` cuts token waste 50–70%**
Community reports that properly configured `.claudeignore` (excluding build artifacts, node_modules, generated files) dramatically reduces context consumption per session.
*Source: [YouTube — top 10 tips guide](https://www.youtube.com/watch?v=T5jylUte3J8) — Community-reported*

**Use `/compact` instead of `/clear`**
`/compact` compresses context while retaining key information; `/clear` wipes everything. For long sessions, `/compact` preserves continuity while freeing tokens.
*Source: [YouTube — top 10 tips guide](https://www.youtube.com/watch?v=T5jylUte3J8) — Community-reported*

## 🧭 Bottom Line

**Most important update:** The source code leak. Not because of what it broke (nothing, operationally), but because it revealed Anthropic's roadmap in unprecedented detail — multi-agent orchestration, cloud-offloaded planning, and anti-distillation measures are all built and waiting behind feature flags.

**Most useful workflow improvement:** The `defer` permission decision in PreToolUse hooks (v2.1.89). This unlocks real production CI/CD patterns with human-in-the-loop approval gates for headless Claude Code sessions.

**Most important model-related change:** No new model this week, but the **1M context beta retirement deadline (April 30)** for Sonnet 4/4.5 is the action item. If you're on older models, migrate now.

**Overhyped / low-signal:** The Companion Pet System from the leak got disproportionate attention. It's a retention gimmick with a Tamagotchi, not a workflow feature. Also, the "Undercover Mode" discourse is mostly about optics — it's dead-code-eliminated in external builds and doesn't affect your usage.

**What developers should try now:**
1. Update to v2.1.92 (`npm update -g @anthropic-ai/claude-code`)
2. Set up `defer` hooks if you run headless pipelines
3. Try the Bedrock wizard if you've been manually configuring AWS
4. If you use OpenClaw, claim your transition credit before April 17 and set a spending cap
5. Run `/powerup` to see what you've been missing
