## 2. Practical Engineering Insights

The LangChain survey and multiple engineering reports converge on several patterns that have hardened this month:

**Orchestration is the new architecture decision.** Three dominant patterns: supervisor hierarchies (LangGraph), role-based crews (CrewAI), and handoff protocols (OpenAI Swarm/A2A). The choice depends on whether your workflows are hierarchical, collaborative, or sequential. Don't default to multi-agent when a single agent with good tool access suffices. ([47billion.com](https://47billion.com/blog/ai-agents-in-production-frameworks-protocols-and-what-actually-works-in-2026/); [langchain.com](https://www.langchain.com/state-of-agent-engineering))

**Evaluation is the bottleneck, not model capability.** 52% of teams now run evals, but only 44.8% do online (production) evals. The emerging standard is a three-layer approach: unit tests (deterministic), integration tests (LLM-as-judge at 53.3%), and adversarial evals (prompt injection, edge cases). Human review remains essential for high-stakes decisions (59.8%). Teams that evaluate systematically see 6x higher deployment success. ([langchain.com](https://www.langchain.com/state-of-agent-engineering); [databricks.com](https://www.databricks.com/blog/enterprise-ai-agent-trends-top-use-cases-governance-evaluations-and-more))

**Observability is table stakes.** 89% of production teams have it. Monitor three signals minimum: schema compliance, tool call success rate, and latency per agent node. One report cited a €200K hallucination that went undetected for days due to missing observability. ([langchain.com](https://www.langchain.com/state-of-agent-engineering); [youtube.com — Zero to Production](https://www.youtube.com/watch?v=cS-CXkA8XYw))

**Multi-model is the default.** 75%+ of production teams use multiple providers, routing by complexity, cost, and latency. OpenAI dominates (>67%) but is rarely exclusive. 57% avoid fine-tuning due to cost; RAG + prompt engineering remains the primary customization path. ([langchain.com](https://www.langchain.com/state-of-agent-engineering))

**Memory is immature but critical.** A 9-type memory taxonomy is emerging, but production challenges include memory poisoning, drift, conflicts, and cache staleness. Treat memory as a first-class architectural concern, not an afterthought. ([youtube.com — Zero to Production](https://www.youtube.com/watch?v=cS-CXkA8XYw); [towardsai.net](https://pub.towardsai.net/ai-agents-in-2026-a-practical-guide-918239017060))

**Cost shifted from #1 to #2 concern.** Quality (32%) now outranks cost as the top production barrier. This suggests model pricing is stabilizing, but reliability engineering is where teams are struggling. ([langchain.com](https://www.langchain.com/state-of-agent-engineering))

## 3. Governance, Risk, and Security

**The adoption-security gap is alarming.** 81% of teams are past the planning phase for agentic AI, yet only 14.4% have full security approval. 88% of organizations reported confirmed or suspected agent-related security incidents in the past year. Only 29% consider themselves security-ready. ([gravitee.io](https://www.gravitee.io/blog/state-of-ai-agent-security-2026-report-when-adoption-outpaces-control); [helpnetsecurity.com](https://www.helpnetsecurity.com/2026/02/23/ai-agent-security-risks-enterprise/); [kiteworks.com](https://www.kiteworks.com/cybersecurity-risk-management/agentic-ai-attack-surface-enterprise-security-2026/))

**Top threat vectors** (per OWASP and incident reports):
- **Prompt injection** — malicious instructions in inputs hijack agent logic; scales with deployment volume
- **Excessive agency** — over-permissioned agents executing unauthorized actions (25%+ of analyzed agent "skills" had vulnerabilities)
- **Data exfiltration** — via shadow AI, transitive API calls, or compromised tools
- **Supply chain attacks** — poisoned training data, tampered models, compromised plugins (47-enterprise incident)
- **MCP protocol vulnerabilities** — hidden injections through tool integrations

([stellarcyber.ai](https://stellarcyber.ai/learn/agentic-ai-securiry-threats/); [genai.owasp.org](https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/); [zenity.io](https://zenity.io/resources/white-papers/2026-threat-landscape-report))

**Governance frameworks to adopt now:**
- **OWASP Top 10 for Agentic Applications** as threat baseline
- **Model AI Governance Framework v1.0** for operational governance
- **Miro AADLC** for lifecycle process
- **NIST AI RMF + ISO/IEC 42001** for compliance mapping

**Team structure:** Cross-functional councils (AI, data, privacy, security, business) with RACI-based accountability outperform centralized-only or fully-embedded models. The "Digital Contractor" mental model — treating agents like contractors who need onboarding, access controls, supervision, and performance reviews — is a useful framing. ([ewsolutions.com](https://www.ewsolutions.com/agentic-ai-governance/); [miro.com](https://miro.com/blog/agent-automation-development-lifecycle/))

**Build vs. buy:** Hybrid dominates. Enterprises favor open frameworks (LangGraph, CrewAI, AutoGen) for custom builds with governance hooks, supplemented by vendor platforms for specific use cases. Pure vendor lock-in is declining as a strategy. ([lovelytics.com](https://lovelytics.com/post/state-of-ai-agents-2026-lessons-on-governance-evaluation-and-scale/))

### Dependencies to Build First
1. **Data access layer** — agents need clean, authorized access to enterprise systems
2. **Identity and access management** — least-privilege for non-human identities
3. **Evaluation infrastructure** — automated + human eval pipelines before scaling

**What separates scalable use cases from demos:** Structured inputs/outputs, clear success metrics, bounded action spaces, existing data pipelines, and human escalation paths. If you can't define when the agent should stop and ask a human, it's not production-ready.

## 6. Emerging Patterns

**Pattern 1: Agent orchestration is moving into the model layer.** Anthropic's native agent teams, Grok's internal multi-agent debate, and Google's A2A protocol all point the same direction. Implication: framework-level orchestration may become less important over time. Don't over-invest in complex custom orchestration that models will handle natively within 12 months.

**Pattern 2: Security is the binding constraint, not capability.** Every data source confirms the same gap: adoption is outpacing security by a wide margin (81% deploying vs. 14.4% security-approved). The 88% incident rate is not a scare statistic — it reflects real operational risk. The teams that invest in security now will scale; the ones that don't will have incidents that freeze deployment org-wide.

**Pattern 3: Governance predicts success.** Databricks' finding that unified governance correlates with 10x more production deployments is the strongest signal in this briefing. Governance isn't overhead — it's the enabler of scale.

**Pattern 4: The "good enough" model tier is expanding.** Open-weight models (DeepSeek V4, GLM-5, Qwen 3.5) are now competitive on agent benchmarks. This creates real optionality for cost-sensitive or data-sensitive workloads. The multi-model routing pattern (75%+ of production teams) will accelerate.

**Overhyped:** Fully autonomous multi-agent systems handling complex cross-departmental workflows without human oversight. The technology is improving, but the governance, security, and reliability infrastructure isn't there yet for most organizations.

**Underhyped:** Single-agent systems with good tool access, strong evaluation, and human-in-the-loop for bounded, high-volume tasks. These are delivering real ROI today with manageable risk.

## 7. What This Means for Our Team

### Skills to Build Now
- **Evaluation engineering** — designing eval suites, LLM-as-judge pipelines, adversarial testing. This is the scarcest and highest-leverage skill.
- **Agent security** — prompt injection defense, least-privilege design, supply chain auditing. Hire or upskill from AppSec.
- **Orchestration literacy** — hands-on experience with LangGraph, CrewAI, and at least one vendor platform (Microsoft Agent Framework or Google ADK depending on our stack).
- **Observability and ops** — extending existing monitoring to cover agent-specific signals (tool call success, schema compliance, action traces).

### Short-Term Execution (0–3 months)
- Pick 1–2 bounded, high-volume use cases (customer support triage, IT alert triage) and build end-to-end with full eval and observability.
- Stand up a governance intake process using AADLC or similar — even a lightweight version prevents shadow AI proliferation.
- Conduct a security assessment using OWASP Top 10 for Agentic Applications as the checklist.

### Longer-Term Organizational Design (3–12 months)
- Establish a central platform team that owns shared infrastructure (eval, observability, identity, governance) while embedded engineers build domain-specific agents.
- Build a multi-model routing capability to avoid vendor lock-in and optimize cost/quality tradeoffs.
- Invest in data access infrastructure — agents are only as good as the systems they can reach.

## Actionable Recommendations

### 3 Things to Evaluate Now
1. **Microsoft Agent Framework** (if on Azure/M365) or **Google Agent Development Kit** (if on GCP) — these are becoming the default enterprise paths and may already be in your licensing.
2. **Open-weight models (DeepSeek V4, GLM-5)** for sensitive workloads where data cannot leave your environment. Benchmark against your actual use cases, not public leaderboards.
3. **Your current MCP and plugin integrations** for supply chain risk. The 47-enterprise credential harvesting incident is a wake-up call.

### 3 Actions to Take Now
1. **Ship one agent to production this quarter** with full eval, observability, and human-in-the-loop. Bounded scope, real users, measurable outcomes. Learn by doing, not by planning.
2. **Adopt the OWASP Top 10 for Agentic Applications** as your security baseline and run a gap assessment against it within 30 days.
3. **Establish a lightweight governance intake process** (request → risk assessment → approval → monitoring) before agent projects proliferate across departments.

### 2 Risks to Prepare For
1. **A security incident that freezes all agent work.** With 88% of organizations reporting incidents and only 14.4% having security approval, this is a when-not-if scenario. Have an incident response plan and a communication strategy ready.
2. **Platform vendor lock-in.** As Salesforce, Microsoft, and Google embed agents into their platforms, switching costs will rise fast. Design for portability now — abstract model calls, use open protocols, own your evaluation data.

### 1 Strategic Opportunity We May Be Underestimating
**Evaluation infrastructure as a competitive moat.** The Databricks data (6x success rate with systematic evals) and LangChain data (quality as #1 barrier) both point to the same conclusion: the teams that build robust, automated evaluation pipelines will deploy more agents, faster, with fewer failures. This isn't glamorous work, but it's the single highest-leverage investment we can make. Every agent we build will benefit from it. Every governance conversation will be easier with it. Start here.
